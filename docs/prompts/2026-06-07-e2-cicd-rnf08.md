## Decisiones (confirmadas por el humano)

- **CodeDeploy** (no SSH): lo nombra el enunciado. SSH queda documentado como plan B.
- **ECR privado** (no público): ya estaba cableado y andando en E1; migrar a público era re-trabajo sin beneficio.
- **OIDC** (no access keys): GitHub asume un rol IAM; cero credenciales de larga vida en el repo.
- **Gate de calidad reusando `ci.yml`**: si falla lint/test/build, no se despliega; sin duplicar pasos.

## Tradeoffs

| Tema | Elegido | Costo |
|---|---|---|
| CodeDeploy vs SSH | Cumple literal el enunciado | Más setup en AWS (agente + App + DG + roles) |
| ECR privado | Reusa infra E1 | La EC2 hace login a ECR (ya resuelto con instance profile) |
| `prune -af` en cada deploy | Evita que se llene el disco (las imágenes por SHA se acumulaban → llenaron la t3.micro) | Re-descarga si se necesita una imagen vieja |
| `workflow_dispatch` | Redeploys/rollbacks manuales | Solo visible una vez que el workflow está en `main` |
